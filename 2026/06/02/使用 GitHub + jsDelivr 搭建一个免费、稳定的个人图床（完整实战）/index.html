使用 GitHub + jsDelivr 搭建一个免费、稳定的个人图床（完整实战）

Posted on 2026-06-02 In Blog Views: Word count in article: 1.8k Reading time ≈ 2 mins.
本文介绍了一种基于 GitHub 公共仓库与 jsDelivr CDN 的免费图床方案，适合 Hexo / GitHub Pages 博客长期、稳定地管理图片资源。

在使用 Hexo / GitHub Pages 写工程类博客时，图片管理往往是最容易被忽略、但后期成本最高的一环：

  • 图片越来越多，博客仓库体积膨胀
  • 国内访问不稳定
  • 免费图床存在跑路风险
  • 后期迁移成本极高

本文记录一次 从 0 开始、可长期使用 的图床搭建方案：

GitHub 公共仓库 + jsDelivr CDN
免费 · 稳定 · 不依赖第三方图床 · 适合工程博客

方案整体结构

最终推荐的结构如下：

Hexo 博客仓库（只放 Markdown）
└─ burroblog-src-next8

GitHub 图床仓库（只放图片）
└─ burro-imgbed
└─ images
└─ blog
└─ xxx.png

图片通过 jsDelivr CDN 访问：

博客仓库与图片仓库彻底解耦，是后期维护最省心、最稳妥的做法。

新建 GitHub 图床仓库

在 GitHub 新建一个仓库：

  • 仓库名：burro-imgbed
  • 可见性：Public（必须）
  • 不需要 README / License

⚠️ jsDelivr 只能加速 公开仓库，图床仓库必须是 Public。

该仓库 只用于存放图片，不放代码、不放文档。

创建图床目录结构（关键步骤）

GitHub 不支持直接创建空文件夹，因此使用 .keep 文件占位。

操作步骤

  1. 进入图床仓库
  2. 点击 Add file → Create new file
  3. 在 文件名输入框 中填写：images/blog/.keep

文件内容保持为空即可。

提交 .keep（GitHub 新 UI）

在 GitHub 新版界面中，提交信息被收进按钮里。

  1. 点击右上角 Commit changes…
  2. 填写提交信息：

Commit message
Create .keep

Extended description（可选）
init image bed structure

然后提交。

提交说明仅用于记录操作，不影响功能。

确认目录创建成功

提交完成后，仓库中应看到：

burro-imgbed
└─ images
└─ blog
└─ .keep

看到 blog/ 目录，说明图床的基础结构已经搭建完成。

上传第一张图片

进入 images/blog/ 目录：

  1. 点击 Add file → Upload files
  2. 上传一张测试图片，例如：test.png

提交即可。

是否可以继续新建子目录？

可以，而且非常推荐。

例如：

images/blog/hexo/
images/blog/diagrams/
images/blog/notes/

创建方式相同：images/blog/hexo/.keep

重要使用规范（强烈建议）

1️⃣ 不要覆盖同名图片

jsDelivr 有缓存，更新图片请 更换文件名。
推荐：hexo-arch-202606.png
不推荐：arch.png

2️⃣ 图床仓库只放图片

不放代码、不放压缩包。

3️⃣ 博客仓库不再存图片

Markdown 与图片彻底解耦，维护成本最低。

✅ 免费
✅ 稳定
✅ 不依赖第三方图床
✅ 不占博客仓库体积
✅ 适合长期维护
