使用 PicGo 实现一键截图 → 上传 → 自动生成图床链接（GitHub + jsDelivr）

Posted on 2026-06-02 In Blog Views: Word count in article: 1.7k Reading time ≈ 2 mins.
本文介绍了如何通过 PicGo 将截图自动上传到 GitHub 图床，并生成可直接使用的 jsDelivr 链接，从而实现高效、稳定的博客图片管理流程。

在上一篇文章中，我已经搭建好了基于 GitHub 公共仓库 + jsDelivr CDN 的免费图床方案。
但在实际写博客时，如果每次都要：

  • 手动截图
  • 打开 GitHub
  • 上传图片
  • 复制链接

这个流程仍然偏重。

本文将进一步优化这一流程，目标是：

截图 → 自动上传 → 自动生成 Markdown 图片链接

最终实现 几乎“无感”的图片插入体验。

为什么需要 PicGo？

PicGo 是一款 开源的图片上传工具，支持多种图床后端，包括：

  • GitHub
  • GitLab
  • 七牛云 / 阿里云 OSS
  • 自定义接口

在本方案中，PicGo 的作用是：

  • 自动把图片上传到 GitHub 图床仓库
  • 自动生成 jsDelivr 访问链接
  • 自动复制 Markdown 图片语法到剪贴板

安装 PicGo

1️⃣ 下载 PicGo

官方项目地址：
https://github.com/Molunerfinn/PicGo

根据系统下载对应版本：

  • Windows：.exe
  • macOS：.dmg
  • Linux：AppImage / deb

安装完成后启动 PicGo。

准备 GitHub Token（必须）

PicGo 通过 GitHub API 上传图片，需要一个 Personal Access Token。

1️⃣ 创建 Token

在 GitHub 中：

  1. 进入 Settings → Developer settings
  2. 打开 Personal access tokens
  3. 选择 Tokens (classic)
  4. 点击 Generate new token

2️⃣ Token 权限设置

至少勾选：

✅ repo

生成后复制 Token（只显示一次）。

配置 PicGo（GitHub 图床）

打开 PicGo，进入 图床设置 → GitHub，填写如下信息：

Repo: Burro-CMD/burro-imgbed
Branch: main
Token: <你的 GitHub Token>
Path: images/blog/
Custom URL: https://cdn.jsdelivr.net/gh/Burro-CMD/burro-imgbed

说明：

  • Repo：你的图床仓库
  • Path：图片上传到的子目录
  • Custom URL：用于生成 jsDelivr 链接

保存配置，并将 GitHub 设置为默认图床。

配置输出为 Markdown 格式（强烈推荐）

在 PicGo 设置中：

  1. 打开 设置 → 设置 Server
  2. 勾选 上传后自动复制链接
  3. 链接格式选择：

生成格式示例：

一键截图 → 自动上传 → 自动生成链接

推荐使用方式

  • Windows：截图工具 / Snipaste
  • macOS：系统截图
  • Linux：系统截图工具

截图完成后：

  1. 复制图片到剪贴板
  2. 点击 PicGo 的 Upload
  3. PicGo 自动完成：
    • 上传图片
    • 生成 jsDelivr 链接
    • 复制 Markdown 语法

你只需要粘贴即可。

常见问题与实践建议

1️⃣ 图片覆盖问题

jsDelivr有缓存机制，不建议覆盖同名文件。
推荐开启PicGo的时间戳重命名，或使用如下命名策略：
hexo-arch-20260602-1530.png

2️⃣ 目录规划建议

推荐在 PicGo 中按内容拆分路径，例如：
images/blog/hexo/
images/blog/diagrams/
images/blog/notes/

不同文章可使用不同 Path。

3️⃣ Token 安全

  • Token 不要提交到 GitHub
  • 不要截图公开
  • 如泄露，立即删除并重新生成

最总效果总结

完成以上配置后，图片插入流程将变为：
截图 → Ctrl+V → 上传 → 粘贴 Markdown

无需再手动操作 GitHub 页面。

总结

通过 PicGo + GitHub 图床 + jsDelivr CDN，可以实现一套：

  • ✅ 免费
  • ✅ 稳定
  • ✅ 高效率
  • ✅ 适合长期维护

的图片管理方案。对于 Hexo / GitHub Pages / NexT 等工程型博客，这是目前性价比极高的一种实践方式。
