Github图床删除图片

Posted on 2026-06-03 In Blog Views: Word count in article: 826 Reading time ≈ 1 mins.
本文主要记录使用PicGo上传了错误的照片，怎样在Github端删除照片的方法。

结论

删除图片 = 从 GitHub 仓库里删除文件。
PicGo 只是“上传工具”，真正的图片存储在 GitHub 仓库。

✅方法一（最简单，推荐）：GitHub 网页直接删 ✅

这是最常用、最安全的方式。
操作步骤

打开图床仓库
https://github.com/Burro-CMD/burro-imgbed

进入图片所在目录
images/blog/xxx.png

点击图片右上角的 🗑 Delete this file
填写 commit message（随便写）
点击 Commit changes

✅ 图片立刻从仓库删除
✅ 不影响其他图片
✅ 不需要本地环境

✅ 方法二：本地 git 删除（适合批量）

如果你本地 clone 了图床仓库：

删除单张

批量删除

✅ 适合一次删很多图
✅ 操作可追溯（Git 历史）

✅ 方法三：PicGo 能删吗？（重要说明）

❌ PicGo 本身不支持删除远端图片

原因很简单：

PicGo 是上传工具
没有“远端文件管理 UI”

👉 删除一定要回到 GitHub 仓库

⚠️ 非常重要：jsDelivr 缓存问题

情况说明
即使你已经从 GitHub 删了图片：

可能：

✅ GitHub 上已 404

⚠️ jsDelivr 短时间内仍能访问

这是 CDN 缓存，不是你删错。

✅ 正确认知

✅ 图片已经“逻辑删除”
✅ CDN 缓存会自动过期
✅ 删图本身是成功的
