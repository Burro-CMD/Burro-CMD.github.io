如何实现两台电脑都可以推送博客

Posted on 2026-05-22 In Blog Views: Word count in article: 894 Reading time ≈ 1 mins.
通过为两台电脑配置SSH key，实现多台电脑推送博客的方法。

背景

  起因是需要在home pc或这work pc上写技术总结。

现象

  在home pc上推送没有问题，但在work pc上总是推送失败。

原因

  两边都有已经部署了环境，但是home pc已经绑定密钥，而work pc没有。也就是说每一台电脑都要进行单独认证，Github的SSH key认证是基于设备+私钥，也就是说:

  • SSH key 存在于你这台电脑：
  • Github 只认你上传的公钥
  • 不会自动同步到其他电脑
  • 每台电脑都要自己的SSH key
  • Github 需要一一登记这些钥匙

现在是work pc没有SSH key，所以被拒绝

解决办法

推荐方法: 每台电脑各自生成一套SSH key， 然后全部添加到Github

Setp 1：生成key

Step 2：复制公钥

Step 3：添加到Github

  1. 打开 https://github.com/settings/keys
  2. 点击 NEW SSH key
    Title：work-pc
    Key: 粘贴

Step 4: 测试

此时返回：

  1. Hi 用户名! —— GitHub 已识别你账（用户名），说明 key 已经绑定成功。
  2. You’ve successfully authenticated —— 核心信息，SSH 认证成功，GitHub 已信任这台公司电脑。
  3. but GitHub does not provide shell access —— 是 GitHub 的正常提示，GitHub 不给你远程 shell（不像服务器那样登录），完全不影响 git/hexo 使用。

Step 5: 再deploy
